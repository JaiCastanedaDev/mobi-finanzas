# Icono de la App Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Reemplazar el icono genérico de la plantilla de Expo (una "A" azul sobre grilla) por un icono propio de mobi-finanzas: una alcancía blanca sobre fondo verde de marca, en todas las variantes de plataforma (universal, Android adaptativo, favicon web, splash) y actualizar `app.json` para usarlas.

**Architecture:** Un SVG maestro (fondo + el path exacto del icono `PiggyBank` de `lucide-react-native`, ya dependencia del proyecto) se parametriza por tamaño/color/escala y se rasteriza a 6 archivos PNG con `sharp`, ejecutado desde un proyecto Node temporal fuera del repo (no toca `package.json` del proyecto). Los PNG resultantes reemplazan los assets actuales en `assets/images/`; luego `app.json` se actualiza para apuntar a las rutas/colores correctos y se elimina el bundle iOS "Icon Composer" sin uso (`assets/expo.icon/`).

**Tech Stack:** Node.js + `sharp` (rasterización SVG→PNG, instalado temporalmente fuera del repo), Expo config (`app.json`).

## Global Constraints

- Color de fondo del icono: `#267b4c` (verde primario de la marca, ver `lib/theme.ts`).
- Color del glyph: `#ffffff` (blanco, coincide con `onPrimary` en `lib/theme.ts`).
- Glyph: el path SVG exacto del icono `PiggyBank` de `lucide-react-native` (no redibujar a mano). Los tres paths, copiados de `node_modules/lucide-react-native/dist/esm/icons/piggy-bank.mjs`, viewBox original `0 0 24 24`:
  1. `M11 17h3v2a1 1 0 0 0 1 1h2a1 1 0 0 0 1-1v-3a3.16 3.16 0 0 0 2-2h1a1 1 0 0 0 1-1v-2a1 1 0 0 0-1-1h-1a5 5 0 0 0-2-4V3a4 4 0 0 0-3.2 1.6l-.3.4H11a6 6 0 0 0-6 6v1a5 5 0 0 0 2 4v3a1 1 0 0 0 1 1h2a1 1 0 0 0 1-1z`
  2. `M16 10h.01`
  3. `M2 8v1a2 2 0 0 0 2 2h1`
- Atributos de trazo del glyph (idénticos a los defaults de lucide): `fill="none" stroke-width="2" stroke-linecap="round" stroke-linejoin="round"`.
- No se modifica `package.json`/`package-lock.json` del proyecto — `sharp` se instala en un directorio temporal fuera del repo.
- No hay infraestructura de testing automatizado para assets estáticos; la verificación es inspección visual de cada PNG generado (con la herramienta de lectura de imágenes) más una comprobación de dimensiones.

- [ ] **Step 4: Escribir el script de generación**

Crear el archivo `/tmp/mobi-icon-gen/generate-icons.js` con este contenido exacto:

```js
const sharp = require('sharp');
const fs = require('fs');
const path = require('path');

const OUT_DIR = process.argv[2];
if (!OUT_DIR) {
  console.error('Uso: node generate-icons.js <directorio-de-salida>');
  process.exit(1);
}

const PIGGY_PATHS = [
  'M11 17h3v2a1 1 0 0 0 1 1h2a1 1 0 0 0 1-1v-3a3.16 3.16 0 0 0 2-2h1a1 1 0 0 0 1-1v-2a1 1 0 0 0-1-1h-1a5 5 0 0 0-2-4V3a4 4 0 0 0-3.2 1.6l-.3.4H11a6 6 0 0 0-6 6v1a5 5 0 0 0 2 4v3a1 1 0 0 0 1 1h2a1 1 0 0 0 1-1z',
  'M16 10h.01',
  'M2 8v1a2 2 0 0 0 2 2h1',
];

function buildSvg({ size, background, stroke, scale, offset }) {
  const bg = background ? `<rect width="${size}" height="${size}" fill="${background}"/>` : '';
  const glyph = stroke
    ? `<g transform="translate(${offset},${offset}) scale(${scale})" fill="none" stroke="${stroke}" stroke-width="2" stroke-linecap="round" stroke-linejoin="round">${PIGGY_PATHS.map(
        (d) => `<path d="${d}"/>`,
      ).join('')}</g>`
    : '';
  return `<svg xmlns="http://www.w3.org/2000/svg" width="${size}" height="${size}" viewBox="0 0 ${size} ${size}">${bg}${glyph}</svg>`;
}

const ASSETS = [
  { file: 'icon.png', size: 1024, background: '#267b4c', stroke: '#ffffff', scale: 25, offset: 212 },
  { file: 'android-icon-foreground.png', size: 432, background: null, stroke: '#ffffff', scale: 11, offset: 84 },
  { file: 'android-icon-background.png', size: 432, background: '#267b4c', stroke: null, scale: 0, offset: 0 },
  { file: 'android-icon-monochrome.png', size: 432, background: null, stroke: '#ffffff', scale: 11, offset: 84 },
  { file: 'favicon.png', size: 192, background: '#267b4c', stroke: '#ffffff', scale: 4.6875, offset: 39.75 },
  { file: 'splash-icon.png', size: 240, background: null, stroke: '#ffffff', scale: 5.859375, offset: 49.6875 },
];

async function main() {
  fs.mkdirSync(OUT_DIR, { recursive: true });
  for (const asset of ASSETS) {
    const svg = buildSvg(asset);
    const outPath = path.join(OUT_DIR, asset.file);
    await sharp(Buffer.from(svg)).png().toFile(outPath);
    console.log('wrote', outPath);
  }
}

main().catch((err) => {
  console.error(err);
  process.exit(1);
});
```

- [ ] **Step 6: Verificar dimensiones de cada PNG generado**

El proyecto principal no tiene `sharp` instalado, así que la verificación se hace desde el script temporal (que sí lo tiene):

```bash
cd /tmp/mobi-icon-gen
node -e "
const sharp = require('sharp');
const fs = require('fs');
const dir = '/d/Personal/mobi-finanzas/assets/images';
const expected = {
  'icon.png': 1024,
  'android-icon-foreground.png': 432,
  'android-icon-background.png': 432,
  'android-icon-monochrome.png': 432,
  'favicon.png': 192,
  'splash-icon.png': 240,
};
(async () => {
  for (const [file, size] of Object.entries(expected)) {
    const meta = await sharp(dir + '/' + file).metadata();
    const ok = meta.width === size && meta.height === size;
    console.log(file, meta.width + 'x' + meta.height, ok ? 'OK' : 'MISMATCH esperado ' + size);
  }
})();
"
```

Expected: las 6 líneas terminan en `OK`. Si alguna dice `MISMATCH`, revisar el `size` correspondiente en `ASSETS` (Step 4) y regenerar (Step 5).
